# وثيقة متطلبات المنتج PRD

## نظام رمسا ERP لإدارة النقل والشحن

**اسم المشروع:** نظام رمسا لإدارة النقل والشحن
**اسم المنشأة:** شركة رمسا للشحن والنقليات
**نوع النظام:** نظام ERP سحابي متكامل
**نطاق التشغيل:** المملكة العربية السعودية
**نوع النشاط:** نقل الطرود والبضائع بين مدن ومناطق المملكة
**المنصات المطلوبة:** لوحة إدارة ويب، بوابة عملاء، تطبيق مندوبين، تطبيق سائقين، واجهات API
**نموذج النظام:** نظام خاص بشركة رمسا، وليس منصة SaaS متعددة الشركات
**اللغات:** العربية والإنجليزية
**العملة:** الريال السعودي
**المنطقة الزمنية:** توقيت المملكة العربية السعودية

---

# 1. الملخص التنفيذي

يهدف المشروع إلى تطوير نظام ERP سحابي متكامل لإدارة جميع عمليات شركة رمسا للشحن والنقليات، بدءًا من تسجيل العميل وإنشاء الشحنة، مرورًا بالاستلام والفرز والنقل بين المدن والمستودعات، وانتهاءً بالتسليم والتحصيل وإغلاق الرحلة ماليًا وتشغيليًا.

يعمل النظام كمركز موحد لإدارة:

* العملاء والعقود والأسعار.
* الشحنات والطرود.
* الفروع والمستودعات.
* الرحلات وخطوط النقل.
* الشاحنات والمركبات.
* السائقين والمندوبين.
* الفرز والتحميل والتفريغ.
* التتبع المباشر للشحنات.
* إثبات الاستلام والتسليم.
* الدفع عند الاستلام والتحصيل.
* الفواتير والمحاسبة.
* المصروفات والعهد والسلف.
* الصيانة والوقود.
* التقارير التشغيلية والمالية.
* التكامل مع الهيئة العامة للنقل ومنصة لوجستي.
* التكامل مع الفوترة الإلكترونية في هيئة الزكاة والضريبة والجمارك.
* التكامل مع العنوان الوطني ومزودي الخرائط والرسائل.

الهدف هو إلغاء الاعتماد على الملفات اليدوية وExcel والرسائل المتفرقة، وتوفير نظام واحد يوضح مكان كل شحنة، والمسؤول عنها، وحالتها، وتكلفتها، وإيرادها، وموعد تسليمها.

# 2. أهداف المشروع

## 2.1 الأهداف التشغيلية

1. توحيد عمليات استقبال الشحنات وإدارتها.
2. معرفة الموقع والحالة الحالية لكل شحنة.
3. تقليل الشحنات المفقودة أو المتأخرة.
4. تنظيم عمليات النقل بين المناطق والفروع.
5. إدارة عمليات الاستلام والتسليم إلكترونيًا.
6. توزيع المهام على السائقين والمندوبين.
7. تحسين سرعة الفرز والتحميل والتسليم.
8. متابعة أداء الفروع والمستودعات والمندوبين.
9. توثيق سلسلة المسؤولية عن الشحنة.
10. تقليل الإدخال اليدوي والأخطاء التشغيلية.

## 2.2 الأهداف المالية

1. احتساب أسعار الشحن تلقائيًا.
2. إدارة عقود وأسعار العملاء التجاريين.
3. متابعة مبالغ الدفع عند الاستلام.
4. مطابقة التحصيلات مع الشحنات.
5. إصدار فواتير إلكترونية.
6. حساب إيرادات ومصاريف كل رحلة.
7. حساب ربحية كل عميل وخط ومنطقة.
8. إدارة المصروفات والوقود والصيانة.
9. إظهار الذمم المدينة والدائنة.
10. توفير تقارير مالية موثوقة للإدارة.

## 2.3 الأهداف النظامية

1. تجهيز النظام للتكامل مع الهيئة العامة للنقل.
2. إدارة بيانات التراخيص وبطاقات التشغيل.
3. إصدار أو تسجيل وثائق النقل الإلكترونية عند توفر الربط الرسمي.
4. الاحتفاظ بسجلات التبادل مع الجهات الحكومية.
5. التحقق من اكتمال العنوان الوطني للشحنات.
6. التكامل مع الفوترة الإلكترونية ومتطلبات زاتكا.
7. توفير سجل تدقيق كامل لجميع العمليات.

## 3.2 بوابة العملاء

## 3.5 واجهات التكامل API

# 5. دورة حياة الشحنة

تمر الشحنة بالمراحل التالية:

1. مسودة.
2. تم إنشاء الشحنة.
3. بانتظار تأكيد الطلب.
4. بانتظار الاستلام من المرسل.
5. تم تعيين مندوب الاستلام.
6. المندوب في الطريق للاستلام.
7. تم استلام الشحنة.
8. وصلت إلى فرع المصدر.
9. تحت الفحص والوزن والقياس.
10. جاهزة للفرز.
11. تم فرزها حسب الوجهة.
12. جاهزة للتحميل.
13. تم تحميلها على الرحلة.
14. غادرت فرع المصدر.
15. في النقل بين المدن.
16. وصلت إلى فرع الوجهة.
17. تم تفريغها.
18. قيد الفرز للتوصيل.
19. تم تعيين مندوب التوصيل.
20. خرجت للتسليم.
21. تم التواصل مع المستلم.
22. تم التسليم بنجاح.
23. تعذر التسليم.
24. مؤجلة بطلب العميل.
25. عنوان غير صحيح.
26. المستلم لا يرد.
27. رفض المستلم.
28. بانتظار إعادة المحاولة.
29. بانتظار الإرجاع.
30. في طريق العودة للمرسل.
31. تم إرجاعها للمرسل.
32. تالفة.
33. مفقودة.
34. تحت التحقيق.
35. ملغاة.
36. مغلقة ماليًا وتشغيليًا.

يجب أن يسجل النظام مع كل تغيير:

* الحالة السابقة.
* الحالة الجديدة.
* المستخدم الذي نفذ التغيير.
* التاريخ والوقت.
* الموقع الجغرافي.
* الفرع أو المستودع.
* الجهاز المستخدم.
* الملاحظات.
* الصور أو المرفقات إن وجدت.
